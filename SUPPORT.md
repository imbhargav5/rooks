# Rooks support

Choose the route that matches your request so it reaches the right workflow.

## Bugs and regressions

Use the [bug report form](https://github.com/imbhargav5/rooks/issues/new?template=bug_report.yml). Include the Rooks, React, and TypeScript versions; entrypoint; minimal reproduction; browser or server-rendering context; and the last version that worked when known.

An external contributor who wants to implement the fix still needs explicit maintainer approval for the proposed scope on the linked issue. See [CONTRIBUTING.md](./CONTRIBUTING.md).

## Usage questions

Use [GitHub Discussions](https://github.com/imbhargav5/rooks/discussions) for help choosing a hook, understanding expected behavior, integrating Rooks with a framework, or troubleshooting application code.

## Features and change proposals

- Use the [feature request form](https://github.com/imbhargav5/rooks/issues/new?template=feature_request.yml) to describe a user problem and proposed behavior.
- If an existing bug or feature issue describes the exact implementation scope, request maintainer approval there. Otherwise, use the [contribution proposal form](https://github.com/imbhargav5/rooks/issues/new?template=contribution_proposal.yml).

Filing or being assigned an issue is not approval. Approval exists only when a maintainer explicitly approves the described scope in the linked issue. Material scope changes need renewed approval.

## Security vulnerabilities

Use [GitHub private vulnerability reporting](https://github.com/imbhargav5/rooks/security/advisories/new). Do not disclose a suspected vulnerability in a public issue or discussion. See [SECURITY.md](./SECURITY.md) for supported versions and reporting details.
